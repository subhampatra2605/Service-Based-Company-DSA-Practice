# 🧩 Binary Search

## 📥 Input

* `nums = [-1,0,3,5,9,12]`
* `target = 9`

---

## 📤 Output

* `4`

👉 Because:

```text id="bs-op1"
nums[4] = 9
```

---

# 🔍 Pattern of the Problem

* **Binary Search / Divide and Conquer**

👉 Core idea:

```text id="bs-core"
Array is sorted
→ eliminate half search space every step
```

---

# 🐢 Brute Force Approach

### Idea:

Traverse array linearly

---

## 💻 Brute Force Code

```java id="bs-bf-code"
class Solution {

    public int search(int[] nums, int target) {

        // Traverse array
        for (int i = 0; i < nums.length; i++) {

            // Target found
            if (nums[i] == target) {
                return i;
            }
        }

        return -1;
    }
}
```

---

## ⏱ Time Complexity

```text id="bs-tc1"
O(n)
```

---

## ❌ Shortcomings

* Searches every element
* Does not use sorted property

---

# ⚡ Optimal Solution (Binary Search)

---

# 💻 Java Code (Line-by-Line Explanation)

```java id="bs-opt-code"
class Solution {

    public int search(int[] nums, int target) {

        // Left boundary
        int left = 0;

        // Right boundary
        int right = nums.length - 1;

        // Continue while valid range exists
        while (left <= right) {

            // Find middle index
            int mid = left + (right - left) / 2;

            // Target found
            if (nums[mid] == target) {

                return mid;
            }

            // Target lies on right side
            else if (nums[mid] < target) {

                left = mid + 1;
            }

            // Target lies on left side
            else {

                right = mid - 1;
            }
        }

        return -1;
    }
}
```

# 🔄 Complete Dry Run

## Input:

```text id="bs-dr1"
nums = [-1,0,3,5,9,12]
target = 9
```

---

# Initial State

```text id="bs-dr2"
left = 0
right = 5
```

---

## Step 1

### Find Mid

```text id="bs-dr3"
mid = 0 + (5-0)/2 = 2
```

### Check

```text id="bs-dr4"
nums[2] = 3
```

```text id="bs-dr5"
3 < 9
```

👉 Target must be on right side

Update:

```text id="bs-dr6"
left = 3
```

---

# Step 2

```text id="bs-dr7"
left = 3
right = 5
```

### Mid

```text id="bs-dr8"
mid = 3 + (5-3)/2 = 4
```

### Check

```text id="bs-dr9"
nums[4] = 9 ✅
```

Return:

```text id="bs-dr10"
4
```

---

# 🎯 Why This Works

👉 Since array is sorted:

```text id="bs-proof1"
Half of array becomes useless after every comparison
```

👉 Search space reduces:

```text id="bs-proof2"
n → n/2 → n/4 → ...
```

---

# ⏱ Complexity

| Type  | Value    |
| ----- | -------- |

# ⚠️ Key Pitfalls

* ❌ Using `mid = (left + right)/2` (overflow risk)
* ❌ Wrong loop condition
* ❌ Infinite loop due to wrong pointer updates
