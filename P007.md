# 🧩 Contains Duplicate

## 📥 Input

* `nums = [1,2,3,1]`

---

## 📤 Output

* `true`

👉 Because:

```text id="cd-op1"
1 appears more than once
```

---

# 🔍 Pattern of the Problem

* **HashSet / Duplicate Detection**

👉 Core idea:

```text id="cd-core"
If element already seen → duplicate exists
```

---

# 🐢 Brute Force Approach

### Idea:

Compare every pair

---

## 💻 Brute Force Code

```java id="cd-bf-code2"
class Solution {
    public boolean containsDuplicate(int[] nums) {

        // Compare every pair
        for (int i = 0; i < nums.length; i++) {

            for (int j = i + 1; j < nums.length; j++) {

                // Duplicate found
                if (nums[i] == nums[j]) {
                    return true;
                }
            }
        }

        return false;
    }
}
```

---

## ⏱ Time Complexity

```text id="cd-tc1"
O(n²)
```

---

## ❌ Shortcomings

* Repeated comparisons
* Slow for large inputs

---

# ⚡ Optimal Solution (HashSet)

---

# 💻 Java Code (Line-by-Line Explanation)

```java id="cd-opt-code2"
import java.util.HashSet;

class Solution {
    public boolean containsDuplicate(int[] nums) {

        // Store unique elements
        HashSet<Integer> set = new HashSet<>();

        // Traverse array
        for (int num : nums) {

            // If already exists → duplicate found
            if (set.contains(num)) {
                return true;
            }

            // Otherwise add to set
            set.add(num);
        }

        return false;
    }
}
```

# 🎯 Why This Works

👉 HashSet stores:

```text id="cd-proof1"

```

👉 Lookup:

```text id="cd-proof2"
O(1)
```

👉 So duplicate detection becomes fast
