# 🧩 Non-Overlapping Intervals

## 📥 Input

* `intervals = [[1,2],[2,3],[3,4],[1,3]]`

---

## 📤 Output

* `1`

👉 Remove:

```text id="noi-op1"
[1,3]
```

👉 Remaining:

```text id="noi-op2"
[1,2] [2,3] [3,4]
```

👉 No overlap remains ✅

---

# 🔍 Pattern of the Problem

* **Greedy + Interval Scheduling**

👉 Core idea:

```text id="noi-core"
Keep interval with smaller ending time
```

---

# 🐢 Brute Force Approach

### Idea:

Try removing each interval and check overlaps

---

## 💻 Brute Force Code

```java id="noi-bf-code"
class Solution {

    public int eraseOverlapIntervals(int[][] intervals) {

        int removeCount = 0;

        // Compare every pair
        for (int i = 0; i < intervals.length; i++) {

            for (int j = i + 1; j < intervals.length; j++) {

                // Overlap condition
                if (intervals[i][1] > intervals[j][0] &&
                    intervals[j][1] > intervals[i][0]) {

                    removeCount++;
                }
            }
        }

        return removeCount;
    }
}
```

---

## ⏱ Time Complexity

```text id="noi-tc1"

```

## ❌ Shortcomings

* Repeated overlap checks
* Does not optimally decide which interval to remove

# ⚡ Optimal Solution (Greedy)

---

# 💻 Java Code (Line-by-Line Explanation)

```java id="noi-opt-code"
import java.util.Arrays;

class Solution {

    public int eraseOverlapIntervals(int[][] intervals) {

        // Step 1: Sort intervals by ending time
        Arrays.sort(intervals,
            (a, b) -> a[1] - b[1]);

        // Store end of previous interval
        int prevEnd = intervals[0][1];

        // Count intervals to remove
        int removeCount = 0;

        // Traverse intervals
        for (int i = 1; i < intervals.length; i++) {

            // Current interval start
            int start = intervals[i][0];

            // Overlap found
            if (start < prevEnd) {

                removeCount++;

            } else {

                // No overlap → update prevEnd
                prevEnd = intervals[i][1];
            }
        }

        return removeCount;
    }
}
```

# 🎯 Why This Works

👉 Interval with smaller end:

```text id="noi-proof1"
Leaves more room for future intervals
```

👉 Greedy choice:

```text id="noi-proof2"
Always keep earliest finishing interval
```

# ⚠️ Key Pitfalls

* ❌ Sorting by start time instead of end time
* ❌ Removing wrong interval
* ❌ Using `<=` instead of `<`
