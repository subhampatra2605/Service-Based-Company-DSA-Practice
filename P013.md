🧩 Koko Eating Bananas

📥 Input

piles = [3,6,7,11]

h = 8



---

📤 Output

4


👉 Because:

If Koko eats 4 bananas/hour:

3 → 1 hour
6 → 2 hours
7 → 2 hours
11 → 3 hours

Total = 8 hours ✅

👉 Smaller speed fails


---

🔍 Pattern of the Problem

Binary Search on Answer


👉 Core idea:

Search minimum valid eating speed


---

🐢 Brute Force Approach

Idea:

Try every eating speed from:

1 → max(piles)


---

💻 Brute Force Code

class Solution {

    public int minEatingSpeed(int[] piles, int h) {

        int max = 0;

        // Find maximum pile
        for (int pile : piles) {
            max = Math.max(max, pile);
        }

        // Try every speed
        for (int speed = 1; speed <= max; speed++) {

            int totalHours = 0;

            // Calculate hours needed
            for (int pile : piles) {

                totalHours += Math.ceil((double)pile / speed);
            }

            // Valid speed found
            if (totalHours <= h) {
                return speed;
            }
        }

        return max;
    }
}


---

⏱ Time Complexity

O(maxPile * n)


---

❌ Shortcomings

Too slow for large piles

Tries every speed unnecessarily



---

⚡ Optimal Solution (Binary Search on Answer)


---

💻 Java Code (Line-by-Line Explanation)

class Solution {

    public int minEatingSpeed(int[] piles, int h) {

        // Minimum possible speed
        int left = 1;

        // Maximum possible speed
        int right = 0;

        // Find largest pile
        for (int pile : piles) {
            right = Math.max(right, pile);
        }

        // Binary search on eating speed
        while (left < right) {

            // Middle speed
            int mid = left + (right - left) / 2;

            int totalHours = 0;

            // Calculate total hours needed
            for (int pile : piles) {

                totalHours += Math.ceil((double)pile / mid);
            }

            // Speed works
            if (totalHours <= h) {

                // Try smaller speed
                right = mid;

            } else {

                // Need faster speed
                left = mid + 1;
            }
        }

        return left;
    }
}

🎯 Why This Works

👉 Eating speed behaves like:

Monotonic function

Meaning:

If speed works,
all larger speeds also work

👉 Perfect for binary search

⚠️ Key Pitfalls

❌ Binary searching indices instead of answers

❌ Using integer division incorrectly
