# 🧩 Assign Cookies

## 📥 Input

* `g = [1,2,3]` → children greed factors
* `s = [1,1]` → cookie sizes

---

## 📤 Output

* `1`

👉 Explanation:

```text id="ac-op1"
Child 1 gets cookie 1 ✅
Child 2 cannot be satisfied ❌
Child 3 cannot be satisfied ❌
```

👉 Total satisfied children:

```text id="ac-op2"
1
```

---

# 🔍 Pattern of the Problem

* **Greedy + Sorting + Two Pointers**

👉 Core idea:

```text id="ac-core"
Give smallest possible valid cookie
to smallest greed child
```

---

# 🐢 Brute Force Approach

### Idea:

For every child → search for unused cookie

---

## 💻 Brute Force Code

```java id="ac-bf-code"
class Solution {
    public int findContentChildren(int[] g, int[] s) {

        boolean[] used = new boolean[s.length];

        int count = 0;

        // Try satisfying each child
        for (int i = 0; i < g.length; i++) {

            for (int j = 0; j < s.length; j++) {

                // If cookie unused and sufficient
                if (!used[j] && s[j] >= g[i]) {

                    used[j] = true;

                    count++;

                    break;
                }
            }
        }

        return count;
    }
}
```

---

## ⏱ Time Complexity

```text id="ac-tc1"
O(n * m)
```

---

## ❌ Shortcomings

* Repeated cookie searches
* Inefficient

# ⚡ Optimal Solution (Greedy + Sorting)

---

# 💻 Java Code (Line-by-Line Explanation)

```java id="ac-opt-code"
import java.util.Arrays;

class Solution {
    public int findContentChildren(int[] g, int[] s) {

        // Step 1: Sort greed factors
        Arrays.sort(g);

        // Step 2: Sort cookie sizes
        Arrays.sort(s);

        // Pointer for children
        int i = 0;

        // Pointer for cookies
        int j = 0;

        // Count satisfied children
        int count = 0;

        // Traverse both arrays
        while (i < g.length && j < s.length) {

            // If current cookie satisfies child
            if (s[j] >= g[i]) {

                count++;

                // Move to next child
                i++;
            }

            // Move to next cookie
            j++;
        }

        return count;
    }
}
```

# 🎯 Why This Works

👉 Sorting ensures:

```text id="ac-proof1"
Small cookies go to small greed children
```

👉 Greedy choice:

```text id="ac-proof2"
Never waste large cookie on small greed child
```

# ⚠️ Key Pitfalls

* ❌ Not sorting arrays
* ❌ Using large cookie unnecessarily
* ❌ Forgetting to move cookie pointer
